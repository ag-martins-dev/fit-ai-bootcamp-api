# Template de Prompt para Criar uma Rota de API

Crie a rota `GET /home/{date}`

## Descrição

Essa rota deve retornar os dados necessários do usuário autenticado para preencher a página inicial que está em https://www.figma.com/design/rm4xc1u8n70ozKSVzHZNxj/FIT.AI--Alunos-?node-id=47-1546&p=f

## Requisitos Técnicos

- Receba uma data no formato YYYY-MM-DD como param da rota.
- Use a biblioteca dayjs para manipular e formatar datas.

## Autenticação

- Rota protegida.

## Response

```ts
interface StatusCode201 {
  activeWorkoutPlanId: string;
  todayWorkoutDay: {
    workoutPlanId: string;
    id: string;
    name: string;
    isRest: boolean;
    weekDay: string;
    estimatedDurationInSeconds: number;
    coverImageUrl?: string;
    exercisesCount: number;
  };
  workoutStreak: number;
  consistencyByDay: {
    [key: string]: {
      workoutDayCompleted: boolean;
      workoutDayStarted: boolean;
    };
  };
}
```

## Regras de Negócio

- A partir da data recebida, calcular o inicio (domingo) e fim (sábado) da semana em UTC.
- Para recuperar o treino do dia, faça o seguinte: 1) Procure pelo workoutPlan ativo; 2) Procure nos dias de treino do workoutPlan ativo o dia correspondente à data atual (recebida na rota).
- workoutStreak: número de dias em sequência que o usuário completou alguma sessão de treino (incluindo dia de descanso). **Exemplo**: se o plano de treino ativo tem segunda, terça, quarta e quinta-feita e o usuário completou todos (incluindo descanso) o streak dele é 4.

**Para montar o `consistencyByDay`**:

- Busque todas as workoutSessions do usuário onde o startedAt está dentro do range domingo 00:00:00 até sábado 23:59:59 da semana correspondente.
- Agrupar as sessões por data (parte da data de startedAt)
- Se uma sessao de treino foi completada (completedAt preenchido), ela deve ser contabilizado tanto em workoutDayCompleted quanto em workoutDayStarted
- Se uma sessao de trino foi apenas iniciada (sem completedAt), ela deve ser contabilizado apenas em workoutDayStarted
